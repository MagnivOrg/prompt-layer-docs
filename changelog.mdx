## February 26, 2026

## Release Notes: nightly-2026-02-26

## Overview

**Release Date:** February 26, 2026
**Previous Release:** Previous day
**Docker Tags:** `nightly-2026-02-26`

This release focuses on enhanced AI assistant capabilities, improved markdown rendering, and better error handling across the platform.

---

## New Features

**Code Execution Support for Google/Vertex AI**
Added native code execution capabilities for Google and Vertex AI models, enabling Python code generation and execution within conversations.
- Models can now generate and execute Python code in sandboxed environments
- Code execution results are preserved and displayed in conversation history
- Supports both `executable_code` and `code_execution_result` content types

**Enhanced Markdown Rendering**
Improved markdown support across the platform with better formatting and display options.
- Rich text formatting now supports tables, code blocks, and nested elements
- Thinking blocks and reasoning content render with improved readability
- Provider-specific metadata preserved in message display

**Computer Agent Improvements**
Enhanced the AI assistant's computer use capabilities with better error handling and status tracking.
- Added human-readable status descriptions for computer agent operations
- Improved error message formatting with actionable context
- Better handling of sandbox agent execution states

---

## Improvements

- Increased OpenSearch batch size from 500 to 1000 documents for better indexing performance
- Added refresh button to sidebar for manual data synchronization
- Improved datetime picker with removable date selection option
- Enhanced evaluation display to show evaluations without scores in prompt analytics
- Fixed Git diff output to show full file paths (300 character width) in deployment summaries
- Added Sentry logging for Centrifugo recovery failures to improve real-time monitoring
- Improved prompt template caching with better error recovery between read and primary database sessions
- Reduced Centrifugo production instances from 4 to 2 for cost optimization
