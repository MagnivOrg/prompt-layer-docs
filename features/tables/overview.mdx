Tables are the workspace surface for running prompt workflows over rows of examples. A Table sheet has:

- **Rows**: examples, requests, test cases, or batch items.
- **Text columns**: editable inputs, labels, expected answers, metadata, or notes.
- **Computed columns**: prompt templates, code, comparisons, assertions, extraction steps, composition, and other runnable work.
- **Sheet controls**: import, export, scoring, analytics, history, row controls, and sheet tabs.

Use this page as the product walkthrough for the Table UI.

## 1. Start a Table

A new Table opens with one sheet, one text column, and one row. The empty state gives you three starting points:

1. **Create from a prompt**: set up prompt inputs, an output column, and evaluation checks.
2. **Import request history**: select logged requests and add them as rows.
3. **Start blank**: build the sheet manually from rows and columns.

## 2. Bring data into the sheet

Use **Upload** when your rows already exist outside the Table.

The upload menu supports:

- **From computer**: upload a CSV file.
- **From request history**: select existing logged requests.

<Frame>
  <img src="/images/tables/polished/02-upload-menu.png" alt="Upload menu with CSV upload and request history import options" />
</Frame>

### Import request history

Choose **From request history** to open the request-history import dialog. Use the date range, search bar, filters, and request grid to narrow the requests before adding them to the sheet.

<Frame>
  <img src="/images/tables/polished/03-request-history-import-full.png" alt="Add from Request History dialog with search, filters, date range, request grid, pagination, and selectable rows" />
</Frame>

After selecting rows, click **Add to Sheet**. Each selected request becomes a row in the Table.

## 3. Export or continue editing

The Table toolbar keeps import and export close to the sheet:

- **Upload** brings in CSV files or request history.
- **Download** exports the current sheet.
- **Tour** opens an in-product walkthrough.
- **Score**, **Analytics**, and **History** open the right-side review tools.

<Frame>
  <img src="/images/tables/polished/04-tour-import-export.png" alt="Table toolbar showing import, export, score, analytics, and history controls" />
</Frame>

## 4. Add columns

Columns define what each row stores or produces. Click **Add Column** to add a new field to the sheet.

Use columns in this order when possible:

1. Input text columns.
2. Prompt or workflow output columns.
3. Comparison, assertion, parsing, extraction, or scoring columns.
4. Notes or reviewer columns.

<Frame>
  <img src="/images/tables/polished/05-add-columns.png" alt="Add Column control in a Table sheet" />
</Frame>

## 5. Use text and computed columns

Text columns hold editable values. Computed columns run work for each row and show an icon for the type of step.

Common computed column types include:

- **Prompt Template** for running a PromptLayer prompt over each row.
- **Code Execution** for custom row-level logic.
- **Composition** for pulling reusable work from another sheet or source column.
- **Comparison**, **Condition**, **Contains**, **Regex**, and numeric distance checks.
- **LLM Assertion** and **AI Data Extraction** for model-judged checks.
- **JSON Path**, **XML Path**, **Regex Extraction**, parsing, count, min/max, coalesce, combine, and diff helpers.

<Frame>
  <img src="/images/tables/polished/06-column-types.png" alt="Table columns showing text inputs and computed prompt, code, quality, and edge-case columns" />
</Frame>

## 6. Configure a computed column

Open a column's menu and choose **Configure column** to edit its settings. Prompt Template columns expose input variables that must be mapped to source columns.

In this example, the Prompt Template column reads values from the `Customer message` column. Source mappings become dependencies, so each row knows which input values to use when the column runs.

<Frame>
  <img src="/images/tables/polished/07-source-mapping.png" alt="Prompt Template column configuration showing input variables mapped to source columns" />
</Frame>

If a variable is unmapped, open the source selector.

<Frame>
  <img src="/images/tables/polished/08-source-selector.png" alt="Source selector for an unmapped Prompt Template input variable" />
</Frame>

Choose the source column for the variable. The mapping is saved as a dependency for that computed column.

<Frame>
  <img src="/images/tables/polished/09-source-assignment.png" alt="Assigning a source column to a Prompt Template input variable" />
</Frame>

## 7. Add rows

Each row is one example to run through the Table. Add rows from the row control at the bottom of the sheet, then fill the input columns.

<Frame>
  <img src="/images/tables/polished/10-add-rows.png" alt="Add row control in a Table sheet" />
</Frame>

## 8. Use multiple sheets

A Table can contain multiple sheets. Use sheets when related workflows need different inputs, columns, or test sets but should live in the same Table.

<Frame>
  <img src="/images/tables/polished/11-multiple-sheets.png" alt="Multiple sheet tabs in a Table" />
</Frame>

## 9. Compose a workflow

Composition columns let one sheet reuse work from another sheet or source column. Use them when you want a workflow to share a reusable intermediate step instead of copying its logic into every sheet.

<Frame>
  <img src="/images/tables/polished/12-composition.png" alt="Composition column configuration that copies values from another source column" />
</Frame>

## 10. Rerun stale work

When an input value or column configuration changes, dependent outputs can become stale. Stale cells are highlighted in amber so you can see what should be rerun before trusting the result.

<Frame>
  <img src="/images/tables/polished/13-stale-rerun.png" alt="A stale computed cell highlighted in amber" />
</Frame>

Use the run controls to update work:

- Run the table to refresh stale or pending work across the sheet.
- Run a computed column from its header to update one output column.
- Stop a running column from the column menu if needed.

<Frame>
  <img src="/images/tables/polished/14-run-table.png" alt="Run controls for refreshing Table work" />
</Frame>

## 11. Score the sheet

Click **Score** when you want one summary signal for the sheet.

<Frame>
  <img src="/images/tables/polished/15-open-scoring.png" alt="Score button in the Table toolbar" />
</Frame>

In the score panel, choose the score columns and scoring mode. Supported modes include **Auto detect**, **Boolean**, **Numeric**, and **Custom code**. Use **Recalculate** after changing the scoring configuration.

<Frame>
  <img src="/images/tables/polished/16-score-configuration.png" alt="Score configuration panel with score columns, scoring mode, and average score" />
</Frame>

## 12. Review history

Click **History** to inspect saved versions and score changes. Use history before and after meaningful edits: imports, prompt changes, column configuration changes, or full reruns.

<Frame>
  <img src="/images/tables/polished/17-open-history.png" alt="History button and version-history walkthrough in a Table" />
</Frame>

## 13. Open analytics

Click **Analytics** to inspect request-level data for Table-generated work. The analytics view includes locked Table and sheet filters, so you can review the requests tied to the current sheet.

<Frame>
  <img src="/images/tables/polished/18-analytics.png" alt="Analytics drawer opened from a Table with locked Table and sheet filters" />
</Frame>
